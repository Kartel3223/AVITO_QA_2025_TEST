
## BUG 1. Кнопка стереть не отображается

**Приоритет:** medium  

**Описание:**  
При вводе ключевого слова для поиска вакансий в поле ввода не отображается кнопка-крестик для быстрого удаления/стирания слова.

**Реальный результат:**  
Иконка крестика не отображается.

**Ожидаемый результат:**  
Иконка крестика отображается корректно и позволяет быстро стереть введённый текст.

**Комментарий по влиянию:**  
Основная функция ввода текста в поиск не сломана. Отсутствие иконки может путать пользователей, но не блокирует использование. Пользователи всё ещё могут стереть текст вручную при помощи клавиатуры.

**Скриншот:**  
![BUG 1. Кнопка стереть не отображается](Task1/pic/bug1.jpg)

## BUG 2. Неверно считается количество вакансий в заголовке блока

**Приоритет:** medium  

**Описание:**  
Заголовок блока «Клиентский сервис» отображает количество доступных вакансий - 2 штуки, но в списке отображается только 1 вакансия.

**Реальный результат:**  
В заголовке блока написано «Клиентский сервис 2», но в списке видна только 1 вакансия.

**Ожидаемый результат:**  
В заголовке должно быть написано «Клиентский сервис 1», поскольку в этом блоке доступна только одна вакансия.

**Комментарий по влиянию:**  
Вводит пользователя в заблуждение, но он всё равно видит фактическое количество вакансий в списке и может перейти к единственной доступной вакансии.

**Скриншот:**  
![BUG 2.  Неверное количество вакансий в заголовке](Task1/pic/bug2.jpg)

## BUG 3. Показано сообщение "Ничего не нашлось" при наличии результатов

**Приоритет:** high  

**Описание:**  
В блоке «Клиентский сервис» при наличии вакансий отображается сообщение «Ничего не нашлось».

**Реальный результат:**  
В блоке «Клиентский сервис» при наличии вакансий всё равно выводится сообщение «Ничего не нашлось».

**Ожидаемый результат:**  
При наличии вакансий в блоке должны отображаться результаты без сообщения «Ничего не нашлось».

**Комментарий по влиянию:**  
Сообщение противоречит фактическому состоянию блока и может заставить пользователя подумать, что вакансий нет, из-за чего он может уйти со страницы.

**Скриншот:**  
![BUG 3. Сообщение «Ничего не нашлось» при наличии вакансий](Task1/pic/bug3.jpg )

## BUG 6. В блоке "Финансы, закупки, аудит" у вакансии отсутствует фильтр поиска

**Приоритет:** high  

**Описание:**  
У вакансии «Финансовый аналитик» отсутствует фильтр «Команда».

**Реальный результат:**  
В карточке вакансии «Финансовый аналитик» не отображается фильтр «Команда».

**Ожидаемый результат:**  
У вакансии «Финансовый аналитик» должен присутствовать фильтр «Команда» наравне с другими вакансиями этого блока.

**Комментарий по влиянию:**  
Отсутствие фильтра может нарушить поиск: вакансия может «выпасть» из результатов фильтрации по команде, пользователи не смогут её найти по заданному фильтру. Это функциональная ошибка. HR или отдел кадров могут тратить время на неподходящих кандидатов, которых не смущает отсутствие информации о команде.

**Скриншот:**  
![BUG 6. Отсутствует фильтр "Команда" у "Финансового аналитика"](Task1/pic/bug6.jpg)

## BUG 10. На странице вакансий перепутаны фильтры между вакансиями

**Приоритет:** high  

**Описание:**  
На странице вакансий в блоке «Аналитика данных» перепутаны фильтры «Команда» у вакансий:  
- «Руководитель аналитики в транзакционное направление жилой недвижимости»  
- «Аналитик данных в бизнес-команду Авито Авто».

**Реальный результат:**  
Указанные вакансии отображаются с некорректно присвоенными фильтрами команд. При фильтрации по командам вакансии появляются не в тех группах, где должны быть.

**Ожидаемый результат:**  
Каждой вакансии должен быть присвоен корректный фильтр команды, соответствующий описанию вакансии. Фильтрация должна отображать эти вакансии в правильных блоках/результатах.

**Комментарий по влиянию:**  
Можно считать баг критическим в контексте логики фильтрации: у двух вакансий некорректно присвоены фильтры, что полностью нарушает логику фильтрации и вводит в заблуждение кандидатов.

**Скриншот:**  
![BUG 10.  Перепутанные фильтры команд между вакансиями](Task1/pic/bug10.jpg )

## BUG 11. В вакансии "Аналитик данных в бизнес-команду Авито Авто" не указан город

**Приоритет:** medium  

**Описание:**  
В блоке «Аналитика данных» в карточке вакансии «Аналитик данных в бизнес-команду Авито Авто» в описании указан только формат работы «офис» без указания города.

**Реальный результат:**  
Отображается формат работы «офис», но не указано, в каком городе расположен офис.

**Ожидаемый результат:**  
В карточке вакансии должен быть указан конкретный город расположения офиса (например, «офис, Москва»/«офис, Санкт-Петербург» и т.п.).

**Комментарий по влиянию:**  
Для соискателей локация работы может быть ключевым фактором при выборе вакансии. Отсутствие города создаёт неопределённость и может привести к тому, что целевые кандидаты пропустят эту вакансию.

**Скриншот:**  
![BUG 11. Не указана локация офиса в вакансии](Task1/pic/bug11.jpg )

## BUG 12. Неправильное склонение слова «вакансия» в подвале

**Описание:**  
В блоке, отображающем количество найденных вакансий (в подвале/счётчике вакансий), используется некорректная форма слова «вакансия».  
При числе 227 отображается «227 вакансии», в то время как грамматически правильно «227 вакансий».

**Реальный результат:**  
При числе 227 отображается текст «227 вакансии».

**Ожидаемый результат:**  
При числе 227 должно отображаться «227 вакансий» (корректное склонение существительного).

**Комментарий по влиянию:**  
Косметический баг: не влияет на функциональность, но негативно влияет на восприятие качества платформы и внимательности к деталям.
